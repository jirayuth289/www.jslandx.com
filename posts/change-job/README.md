## วิธีการแก้ปัญหา

ขอยกตัวอย่าง เช่น เราย้ายงานและต้องทำโปรเจคที่ต้องรับงานต่อจากคนอื่น

1. จัดลำดับความสำคัญเรื่องที่ต้องทำความเข้าใจก่อน
2. Business Objective ต้องมาก่อนเสมอ เพราะการจะเขียนโปรแกรมให้แก้ไขปัญหาธุรกิจ ต้องเข้าใจ context ก่อน ไม่งั้นเราก็จะเขียนออกมาไม่ตรงตามความต้องการ และต้องแก้วนไปมา
3. Skill Collaboration เมื่อเรารู้สึก requirements หรือข้อมูลที่เราได้รับมา มันดูไม่ชัดเจน ไม่เป็นตรรกะ มันดูไม่เป็นผลดีกับตัวเอง กับทีม เราต้องหาวิธีเพื่อ clarifying
4. หลังจากนั้นแล้วค่อยมาเริ่มเขียนโค้ด ถ้ามีการเปลียน Programming Language เราสามารถเรียนเองได้ เพราะสามารถค้นหาได้ทั่วไป


> ประโยคที่ว่า เรื่องทุกเรื่องต้องใช้เวลา กว่าจะทำได้แบบนั้นแบบนี้ อาจะมีช่วงเวลาที่มีความสับสัน หรือ research 

มากกว่านั้นคือต้องทำความเข้าใจหลายเรื่องในเวลาเดียวกัน ในงานพัฒนาซอฟต์แวร์ก็เช่นกันเรื่องหลัก ๆ คือ

- Business Requirement
- Programming Skill

เป็นเรื่องที่ต้องใช้ deep thinking และด้วย deadline จึงต้องมีการสลับ context ทำให้เกิดความเหนื่อยล้า และคุณภาพของความเข้าใจและการตัดสินใจลดลง

ก็ต้องถือว่าเป็นปกติในสายงานอยู่แล้ว เราหลีกเลี่ยงได้ยาก ทักษะที่สำคัญคือการรับมือกับปัญหาเฉพาะหน้า

## วิถีดั้งเดิม

ตามสันชาตญาณความเป็นจริงเราสามารถโฟกัสได้ทีละอย่าง เราไม่มี working memory เพียงพอต่อเรื่องที่ยากหลายหลายเรื่องในเวลาเดียวกันได้อย่างมีประสิทธิภาพ เพราะยังไงเราก็เป็นมนุษย์ เราสามารถโฟกัสได้ทีละอย่าง

ซึ่งบนความเป็นจริงคงไม่มีใครมาสอนให้คุณเข้าใจได้ทั้งหมดได้ หรือถึงแม้จะได้แต่เราก็ไม่สามารถรับความรู้และสามารถตกผลึกได้ในเวลาอันสั้น
